Overview:
This assignment involves designing and building a realistic relational database from scratch using SQLite, on the topic of a hospital management system. Rather than downloading an existing dataset, all data was programmatically generated to simulate realistic hospital records.

Dataset:
The database is built around a hospital setting and contains multiple related tables covering doctors, departments, patients, and appointments.

How to Run It:
Open the .py generation script and run it to recreate the SQLite database file from scratch. The database can then be opened and queried using SQLite directly, DB Browser for SQLite, or any other SQLite-compatible tool.

Conclusion:
The hospital database is designed to effectively organise and manage information about doctors,
departments, patients, and appointment. The database assures data integrity and scalability
by incorporating normalization, foreign keys, and appropriate data types. Privacy and ethical concerns were
prioritized to establish data protection and safeguard sensitive information.
